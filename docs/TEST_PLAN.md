# 测试计划

> 目标：确认无头服务器不仅能启动，还能稳定跨日期、触发特殊事件、
> 保存存档，并在极端天气、地震、节日、断线和 Mod 变更后保持可恢复。

## 测试原则

- 每次只变更一个变量：镜像版本、Mod、配置或存档只改一项。
- 每个特殊日期、特殊天气和 Mod 变更前都先执行 `backup`。
- 先用全新测试存档验证，再迁移真实存档。
- 每轮记录游戏日期、在线玩家、Mod 列表、日志摘要和结论。
- 不要在过夜保存动画、节日结算或剧情切场时强停容器。

## 快速命令

Windows：

```powershell
.\setup.ps1 doctor
.\setup.ps1 login
.\setup.ps1 download
.\setup.ps1 smoke
.\setup.ps1 backup
.\setup.ps1 logs
```

Linux / macOS：

```bash
./scripts/sdv-server.sh doctor
./scripts/sdv-server.sh login
./scripts/sdv-server.sh download
./scripts/sdv-server.sh smoke
./scripts/sdv-server.sh backup
./scripts/sdv-server.sh logs
```

## 基础冒烟测试

1. 执行 `login` 完成 Steam 授权。
2. 执行 `download` 下载或更新游戏文件。
3. 执行 `smoke` 启动服务并检查端口。
4. 打开 `http://localhost:5800`，确认无头房主界面可访问。
5. 客户端加入测试农场。
6. 完成一次睡觉过夜。
7. 执行 `backup` 生成第一份基线存档备份。
8. 执行 `logs`，确认没有 SMAPI 红色异常或崩溃堆栈。

通过标准：

- 玩家可加入。
- 日期可以推进。
- 过夜后存档存在。
- 容器没有异常退出。
- 基线备份已写入 `backups/`。

## 存档完整性检查

每次完成高风险场景后执行：

1. 完成一次过夜保存。
2. 执行 `.\setup.ps1 backup`。
3. 执行 `.\setup.ps1 restart`。
4. 玩家重新加入服务器。
5. 再完成一次过夜保存。
6. 查看最近日志。

重点检查：

- 最近一天的数据没有回档。
- 背包、箱子、作物、建筑、任务状态没有异常丢失。
- 事件解锁状态没有重复触发或消失。
- 日志没有持续刷出 save、XML、content patch 或 SMAPI 异常。

## 极端天气专项

无头服务器在天气日最需要关注切场、音效、闪电、地图物体变化和跨日保存。
下面每个场景都必须先备份，再测试当天，再重启恢复。

### 雷雨和闪电

测试点：

- 雷雨当天玩家加入、退出、重连是否正常。
- 闪电击中农场物体后，地图状态是否同步。
- 当天过夜是否卡在保存或统计界面。
- 重启后避雷针、作物、树木和地面物体状态是否一致。

通过标准：

- 雷雨音效和闪电事件不会导致无头房主卡死。
- 过夜后存档可继续加载。
- 日志没有持续的天气、音频或地图对象异常。

### 绿色雨

绿色雨只在支持该事件的游戏版本中测试。若当前版本或 Mod 环境不支持，
记录为“不适用”，不要强行安装未知 Mod。

测试点：

- 当天进入农场、城镇和室内地图是否正常。
- 特殊植物、杂草或临时地图变化是否同步给玩家。
- 当天 NPC 行程和天气相关剧情是否卡住。
- 过夜后临时天气状态是否正确清理。

通过标准：

- 玩家可以完整度过绿色雨当天。
- 第二天地图状态恢复正常。
- 重启后存档仍可读取。

### 暴雪、异常降雨和天气 Mod

如果安装了天气扩展、季节扩展、地图扩展或时间控制类 Mod，必须额外覆盖：

- 冬季降雪和暴雪效果。
- 多日连续雨天。
- 天气被 Mod 强制切换的当天。
- 天气和节日重叠的日期。

通过标准：

- 天气切换不阻塞时间推进。
- 地图加载、NPC 行程和过夜保存不报错。
- 移除天气 Mod 后，测试存档仍能加载或有明确回滚方案。

## 日期和事件矩阵

### 春 1 到春 5：早期解锁

重点排查：

- 新档创建是否稳定。
- 春 2、春 3、春 4 过夜是否正常。
- 春 5 矿洞相关解锁是否卡住。
- 邮件、天气、电视、NPC 日程是否产生异常日志。

通过标准：

- 春 5 后玩家可正常进入矿区相关区域。
- 无头房主没有被剧情、弹窗或菜单卡死。

### 夏 3：地震 / 铁路开启

测试步骤：

1. 在春 28 过夜前执行 `backup`。
2. 从春 28 睡到夏 1，再继续推进到夏 3。
3. 每天过夜后查看 SMAPI 和容器日志。
4. 夏 3 后让客户端前往铁路区域验证是否开放。
5. 执行 `restart` 后再次加入，确认事件状态已保存。

通过标准：

- 地震触发期间服务器不崩溃。
- 不出现需要人工点击才能继续的阻塞界面。
- 铁路区域状态正确保存。
- 玩家重连后地图状态一致。

重点记录：

- 触发前日期。
- 触发后日期。
- 在线玩家数量。
- 是否有人在过夜动画期间断线。
- 当天是否安装了额外 Mod。

### 节日场景

建议至少覆盖：

- 春 13 复活节。
- 春 24 花舞节。
- 夏 11 夏威夷宴会。
- 夏 28 月光水母之舞。
- 秋 16 星露谷展览会。
- 秋 27 万灵节。
- 冬 8 冰雪节。
- 冬 25 冬日星盛宴。

测试点：

- 节日前一天过夜是否正常。
- 节日当天是否卡在邀请、等待、剧情或离场状态。
- 多人同时进入节日地图是否正常。
- 节日结束后是否返回农场。
- 节日当天重启服务器后是否能恢复。

通过标准：

- 节日不阻塞时间推进。
- 玩家离场后农场状态一致。
- 保存文件没有损坏。

### 随机农场事件

需要长期观察或通过测试 Mod 辅助触发：

- 作物仙子。
- 女巫事件。
- 奇怪胶囊。
- 石猫头鹰。
- 陨石。
- 雷雨和闪电。

通过标准：

- 事件触发后存档可继续加载。
- 农场地图物体变化正常同步给玩家。
- 没有因为无头房主无法点击弹窗而卡死。

## 多人和连接稳定性

### 玩家断线重连

测试步骤：

1. 玩家 A 加入。
2. 玩家 B 加入。
3. 玩家 A 断网或直接退出。
4. 玩家 B 留在线上继续活动。
5. 玩家 A 重连。

通过标准：

- 在线玩家不被踢出。
- 重连后背包、位置、任务状态正常。
- 日志没有持续刷错。

### 过夜时断线

测试步骤：

1. 多名玩家进入睡觉流程。
2. 其中一名玩家在过夜阶段退出。
3. 观察服务器是否继续保存。

通过标准：

- 时间能推进。
- 存档不会重复写坏。
- 退出玩家重连后状态可接受。

## 容器生命周期测试

### 正常重启

```powershell
.\setup.ps1 backup
.\setup.ps1 restart
```

通过标准：

- 重启后可重新加入。
- 最近一次过夜后的存档可读取。
- Steam session 不需要每次重新验证。

### 异常停止

测试步骤：

1. 确认当前不在保存动画或节日结算中。
2. 玩家在线时执行 `docker stop sdv-server`。
3. 再执行 `.\setup.ps1 start`。
4. 客户端重新加入。

通过标准：

- 不应破坏最近一次稳定保存点。
- 如果当天进度丢失，必须能从 `backups/` 回滚。
- 文档明确提醒不要在写存档时强停容器。

## Mod 兼容测试

每次新增 Mod 后执行：

1. 执行 `backup`。
2. 停止服务器。
3. 新增一个 Mod。
4. 启动服务器。
5. 查看 SMAPI 是否报告依赖缺失。
6. 玩家加入并完成一次过夜。
7. 推进至少 3 天。
8. 覆盖一个节日、地震或极端天气日期。
9. 重启服务器后再次加入。

高风险 Mod：

- 修改时间流速的 Mod。
- 修改多人逻辑的 Mod。
- 修改睡觉、保存、菜单或剧情事件的 Mod。
- 修改天气、季节、地图加载和 NPC 日程的 Mod。

通过标准：

- 无持续红色报错。
- 过夜和保存正常。
- 特殊事件触发后不需要人工操作无头主机。

## 长稳测试

建议至少跑：

- 现实时间 24 小时无人值守。
- 游戏内 1 个季节。
- 2 名以上玩家断续加入。
- 至少 1 次容器更新和回滚演练。
- 至少 1 次雷雨或绿色雨。
- 至少 1 次节日。
- 至少 1 次地震或随机农场事件。

记录模板：

```text
测试日期：
镜像版本：
游戏版本：
SMAPI 版本：
Mod 列表：
存档年份 / 季节 / 日期：
在线玩家：
触发事件：
天气：
是否先备份：
是否过夜成功：
是否重启后可读：
日志摘要：
结论：
```

## 阻塞级问题

发现以下情况时不要迁移真实存档：

- 服务器在过夜时崩溃。
- 雷雨、绿色雨、地震、节日或剧情事件后无法继续。
- 存档被写坏或回档不可控。
- 新增 Mod 后无法稳定重启。
- 玩家加入后持续不同步。
- 重启后最近一次稳定保存点无法读取。

- README 快速开始可照做。
- `.env.example` 不包含真实账号或密钥。

- Windows 和 Linux 脚本至少完成语法检查。
- `backup` 命令可正常生成备份文件。
- 基础冒烟测试已通过。
- 极端天气、地震、节日和存档恢复测试已有记录。
